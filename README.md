# Book_my_stay_app
# 🏨 Hotel Booking Management System

### App-Based Learning Using Core Java & Data Structures

## 📌 Project Overview

The **Hotel Booking Management System** is a console-based Java application designed to demonstrate how **Core Java** and **fundamental data structures** can be used to solve real-world software engineering problems.

Instead of treating data structures as isolated theory, this project shows how they support real application behavior such as **request handling, inventory management, and booking validation**.

The system is designed with an **incremental learning approach**, where each feature introduces a new concept while reinforcing previously learned principles.

---

## 🎯 Objectives

This project demonstrates how data structures and Java concepts can be applied to build a practical system that ensures:

* **Fair request handling** using **FIFO principles**
* **Real-time inventory consistency** across booking operations
* **Prevention of double booking** through uniqueness enforcement
* **Extensible and maintainable architecture** suitable for real applications

---

## 🧠 Learning Approach

The application is developed **step-by-step**, similar to how real production systems evolve.

Each stage of the project focuses on:

* Why a **specific data structure** is needed
* What **problem it solves** in a real business scenario
* What **limitations exist** before improving the design

This approach helps learners understand **not only how to use data structures, but when and why to use them**.

---

## 🏗️ Technologies Used

* **Java (Core Java)**
* **Object-Oriented Programming (OOP)**
* **Data Structures**

    * Queue (FIFO request handling)
    * Lists
    * Hash-based structures for uniqueness
* **Console-based interaction**

---

## 📂 Project Scope

The primary focus of the project is on:

* Data structures implementation
* Object-oriented system design
* Clean and maintainable Java architecture
* Real-world problem modeling

To maintain focus on the **core logic**, the project intentionally minimizes:

* Complex UI
* Extensive user input handling

All interactions are **deterministic and console-based**, making the system easier to debug and understand.

## 🚀 Key Features

* Room booking management
* Request handling using queue structures
* Prevention of duplicate bookings
* Simple and traceable console operations
* Modular and extensible code structure
